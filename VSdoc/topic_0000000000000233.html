Remarks
The DataSource should implement either IList, IBindingList, or IListSource. Some common examples are the following types of objects: <list type="unordered"> <item><description>DataView</description></item> <item><description>DataTable</description></item> <item><description>DataSet</description></item> <item><description>DataViewManager</description></item> <item><description>BindingSource</description></item> </list>

When binding to a list container (i.e. one that implements the IListSource interface, such as DataSet) you must also set the DataMember property in order to identify which particular list you would like to display. You may also set the DataMember property even when DataSource refers to a list, since DataMember can also be used to navigate relations between lists.

When a DataSource is set, the control will create OLVColumns to show any data source columns that are not already shown.

If the DataSource is changed, you will have to remove any previously created columns, since they will be configured for the previous DataSource. ObjectListView.Reset.
