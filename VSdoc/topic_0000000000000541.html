Remarks

In this implementation (2009-09), these objects are essentially passive. Setting properties does not automatically change the associated group in the listview. Collapsed and Collapsible are two exceptions to this and give immediate results.

This really should be a subclass of ListViewGroup, but that class is sealed (why is that?). So this class provides the same interface as a ListViewGroup, plus many other new properties.
